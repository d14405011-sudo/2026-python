q10226  排列 + 禁止站位
--------------------------------------------------------------------------------
【easy 版問題】
  - parse_input 使用 try/except 嘗試 T 格式後再回退，邏輯複雜且難以維護。
  - 人名列表以 chr(ord("A") + i) 逐字計算，可讀性低。
  - keep_only_changed_suffix 使用手寫 while 迴圈比較前綴，是 Python 層級迴圈。

【opt 版改進】
  (1) parse_input 改用逐 token 迭代器（iter + next）直接按 T 格式解析，
      移除 70 行的 try/except 自動偵測邏輯，程式縮短約 40 行。
  (2) 人名改用 string.ascii_uppercase[:n] 切片，一目了然。
  (3) keep_only_changed_suffix 改呼叫 os.path.commonprefix，以 C 實作
      取代 Python while 迴圈，在排列數量多時有明顯速度提升。
  (4) list_valid_orders 將 used[i] 與 position in banned[i] 合併為
      單一 continue 條件，減少縮排層次。

q10235  蛇形路徑計數 DP
--------------------------------------------------------------------------------
【easy 版問題】
  - states / nxt / new_row_dp 皆使用普通 dict，每次存取需呼叫 .get(key, 0)。
  - right_candidates / down_candidates 每格都建立一個小 list，累積開銷不低。
  - 下邊 bitmask 設定需要獨立的 if down == 1 分支。

【opt 版改進】
  (1) 所有臨時字典改用 defaultdict(int)，省去所有 .get() 呼叫，
      讀寫語法更簡潔。
  (2) can_right / can_down 旗標在進入格子前預先計算並快取，
      避免在最內層迴圈重複索引 grid 二維陣列。
  (3) 方向候選改用 (0, 1) tuple 配合 continue，消除每格兩個臨時 list 的建立。
  (4) nm = next_mask | (down << c) 一行取代兩行條件賦值。

q10242  SCC + DAG 最長路
--------------------------------------------------------------------------------
【easy 版問題】
  - Kosaraju 演算法的兩次 DFS 採遞迴實作。
    Python 預設遞迴深度限制為 1000；若圖的節點數超過此限，
    程式會拋出 RecursionError 而直接崩潰。
  - DAG 邊集合以 set() 儲存但仍需手動 if cv not in dag[cu] 判斷。

【opt 版改進】
  (1) 第一次 DFS（計算後序）改為使用 iter(adj_list) 保存迭代器狀態的
      迭代版本，完整模擬遞迴後序遍歷，無任何深度限制。
  (2) 第二次 DFS（SCC 標記）改用 stack.pop() 的簡單 DFS，同樣無遞迴。
  (3) DAG 仍以 set 儲存，cv not in dag[cu] 的保護條件保持不變，
      但整體可處理節點數遠超 1000 的大型測試資料。

q10252  曼哈頓距離最小化
--------------------------------------------------------------------------------
【easy 版問題】
  - one_axis 計算最小成本時使用 sum(abs(v - left_mid) for v in values)，
    為線性 O(n) generator，在 n 很大時仍需逐元素計算。
  - 輸入使用 list(map(int, raw.strip().split())) 預先轉換所有 token 為整數。

【opt 版改進】
  (1) one_axis 改用前綴和陣列（prefix sum），以 O(1) 算術公式直接計算：
        left_cost  = (lo+1)*mid - prefix[lo+1]
        right_cost = (prefix[n] - prefix[lo+1]) - (n-lo-1)*mid
      sort 後只需額外一次 O(n) 前綴和建立，後續每個中位數候選的
      成本查詢皆為 O(1)，常數項優於 generator 推導式。
  (2) 輸入改用 iter(raw.split()) 的延遲迭代器，避免預先建立完整整數列表，
      節省一次完整的 map + list 分配。

q10268  雞蛋問題（Egg Drop）
--------------------------------------------------------------------------------
【easy 版問題】
  - 內迴圈中對 reachable[e] 設了 floors 上限截斷：
      if reachable[e] > floors: reachable[e] = floors
    Python 整數無溢位問題，截斷只是多餘操作；
    更糟的是它讓值停留在較小數字，導致外層需要更多輪才能觸發提前結束。
  - 輸入解析用 line.strip().split()，多一次字串操作。

【opt 版改進】
  (1) 移除 floors 截斷，讓 reachable[e] 自然增長至正確的組合數值
      C(moves,1)+C(moves,2)+...+C(moves,eggs)，外層迴圈更早達到
      reachable[eggs] >= floors 的條件並提前返回。
  (2) 輸入解析改用 line.split() + len(parts) < 2 防空行，
      省去 strip() 呼叫並邏輯更清晰。
